# cursor cross-reviewer Phase 1 verdict — M0.5 (2026-05-10)

## Summary

- Blockers: 0
- Majors: 3
- Minors: 2
- Overall verdict: **小改后继续**。M0.5 的方向符合 ADR-016 path C：`cli-runner.ts` 没被重写，CodingDriver adapter / Fake driver / capability manifest 都已落地；但 trace redaction 的对象/数组递归语义、workspace 创建前校验、以及外层 shutdown 5s fallback 与内层 SIGKILL 计时存在隐藏断点。

## 4 focus items 逐条 PASS/FAIL + 一句证据

1. **cli-runner.ts diff 最小化 / 向后兼容 / 不破坏 Eva web/iOS 路径 — PASS**  
   `RunSessionParams.detached` 是 optional，spawn 默认仍是 `false`，只有 `ClaudeCodeDriver` 显式传 `detached: true`；旧 web/iOS 直接 `runSession()` 路径默认不进入 process group kill。

2. **workspace 隔离 robust (symlink escape, realpath validation) — FAIL**  
   已有 `realpathSync(spec.workspace) === wsReal` 能挡 symlink escape，但 `expected = join(workspaceRoot, spec.runId)` 在校验前就 `mkdirSync(expected)`，恶意 `runId` 可先在 workspace root 外创建目录再被 reject。

3. **process group cancel 路径 (SIGTERM 5s SIGKILL) — FAIL**  
   `cli-runner.ts` 内层确实有 SIGTERM → 5s SIGKILL，但 `vessel-core.ts` 外层 hard-exit fallback 也是 5s，可能在内层 SIGKILL timer 触发前直接退出父进程，留下忽略 SIGTERM 的子进程组。

4. **trace 是否会泄露 prompt / stdout — FAIL**  
   stdout 写盘前调用了 `redactPayload()`，方向正确；但 field blacklist 命中对象/数组时只递归、不继承 mask，`payload.message.content[0]` 这类数组字符串和任意 nested body key 仍可能落盘。

## Findings (BLOCKER/MAJOR/MINOR with file:line)

### M1 [MAJOR] Field-path blacklist 对对象/数组子树不继承，可能泄露 prompt / stdout 内容

**Where**: `packages/backend/src/observability/trace-redactor.ts:114`

**Issue**: `redactValue()` 只在当前 path 被 blacklist 且 value 是非 object 时返回 `***`；如果 blacklist 字段本身是 object/array，就继续递归，并把 path 改成 `payload.message.content[0]` / `payload.api_response.body.foo`。这些 descendant path 不再命中 `.content` / `payload.api_response.body`。

**Evidence**: `REDACTED_FIELD_SUFFIXES` 里有 `.content`，但数组元素 path 会变成 `...content[0]`，不以 `.content` 结尾；spec 要求 `payload.api_response.body` 必脱敏为 `***`，不是只脱敏其中某些标准 key。

**Suggested fix**: 给 `redactValue(value, path, inheritedRedacted = false)` 加继承标志；一旦 `isPathRedacted(path)` 命中，整棵子树 leaf 都返回 `***`。如果确实想保留结构，也应对所有 descendant leaf 强制 mask，而不是重新按 descendant path 判断。

### M2 [MAJOR] 外层 5s hard-exit 可能抢在内层 process group SIGKILL 前退出

**Where**: `packages/backend/src/cli/vessel-core.ts:90`, `packages/backend/src/cli-runner.ts:104`

**Issue**: CLI 收到 SIGINT/SIGTERM 后，`vessel-core` 5 秒后 `process.exit()`；同时 `cli-runner` 的 abort handler 也是 5 秒后对 child 或 process group 发 SIGKILL。两个 timer 同预算，调度顺序不保证，父进程可能先退出，导致忽略 SIGTERM 的 `claude` 子进程组没有收到 SIGKILL。

**Suggested fix**: 外层 fallback 留出内层 kill grace，例如 6s 或 7s；或者让 `runIntent`/driver cancel 返回可 await 的 cleanup promise，外层只在 cleanup 超时后退出。

### M3 [MAJOR] workspace 目录在 runId 校验前创建，path traversal 可产生越界副作用

**Where**: `packages/backend/src/drivers/cli-runner-driver.ts:66`

**Issue**: `expected = join(workspaceRoot, spec.runId)` 后立即 `mkdirSync(expected)`，然后才 realpath 校验。若 `spec.runId` 含 `../`，代码会先创建 workspace root 外的目录，再在 `wsReal.startsWith(rootReal + sep)` 处 throw。

**Suggested fix**: 先校验 `runId` 格式（例如 UUID / `^[A-Za-z0-9_-]+$` 且不含 path separator），或先 `resolve(workspaceRoot, spec.runId)` 并确认仍在 `rootReal` 下，再 `mkdirSync()`。

### m1 [MINOR] header blacklist 大小写敏感，和 HTTP header 语义不一致

**Where**: `packages/backend/src/observability/trace-redactor.ts:19`

**Issue**: blacklist 覆盖 `payload.headers.authorization` / `cookie` / `x-api-key`，但 `Authorization` / `Cookie` / `X-API-Key` 不命中 field path。长 token 可能被 pattern 兜住，短 session cookie 或内部 bearer 值仍可能泄露。

**Suggested fix**: 在 `headers` 子树下构造 path 时 lower-case key，或让 `isPathRedacted()` 对 `payload.headers.<key>` 做 case-insensitive 比较。

### m2 [MINOR] FakeCodingDriver 没镜像真实 driver 的 workspace escape 校验，测试会漏掉隔离回归

**Where**: `packages/backend/src/drivers/fake-coding-driver.ts:43`

**Issue**: 注释说 orchestrator 已经 enforce path，但 orchestrator 只是用 `workspaceFor(runId)` 生成路径，真实的 symlink / traversal 防线在 `ClaudeCodeDriver.submit()`。Fake driver 不镜像这条防线，`test-coding-driver.ts` 的隔离测试只能证明文件写进 temp data dir，不能覆盖 symlink escape / `runId` traversal。

**Suggested fix**: 把 workspace validation 提成共享 helper，real driver 和 fake driver/test 都调用；至少新增一个直接测 helper 的 symlink escape / `../` runId case。

## Positive observations (≥2)

- `cli-runner.ts` 的改动保持了 ADR-016 要求的 adapter 路线：`detached` 默认为 false，现有 Eva web/iOS 调用方不需要迁移，也不会默认进入 process group kill。
- `ClaudeCodeDriver` 对 `spec.workspace` 做 realpath equality check，能挡住“传入路径是 symlink 但看起来在 workspace 下”的常见逃逸。
- Trace 文件权限方向正确：trace dir 用 `0700`，stdout/json 文件用 `0600`，并且 `FileTraceWriter.write()` 在入口统一调用 redaction，不完全依赖 caller 自觉。
- abort signal 链主干是接上的：`vessel-core AbortController` → `runIntent.abortSignal` → `SkillContext.abortSignal` → `CodingDriver` 内部 AbortController → `runSession.signal` → `process.kill(-pid)`。

## 异质性确认

本次最像 Claude 主视角不容易立刻看到的细节是 **path 字符串在数组递归时变成 `...content[0]`，导致 `.content` suffix blacklist 失效**。这不是“有没有 redactor”这种显性问题，而是 byte-level path construction 和 spec 语义之间的偏差；同类问题也出现在外层/内层两个 5s timer 的调度竞态上。
