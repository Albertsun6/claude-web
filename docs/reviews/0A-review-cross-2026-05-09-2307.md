## Findings

### M1 [MAJOR] Trace 脱敏目标和 Trace Replay 目标互相冲突

**Where**: `docs/design/REQUIREMENTS.md §C.1 NFR-O2 / NFR-O3`，`docs/design/FRAMEWORK.md §5 Trace 协议`  
**Lens**: 1, 4  
**Issue**: NFR-O2 要求 trace payload 不出现 `user_prompt` / 私人路径 / token-like 字符串，但 NFR-O3 又要求 replay 输出包含“原 prompt + 原 artifact 路径”。  
**Why this matters**: 这会让 M0 trace 实现时不知道该保留原文还是强制脱敏。若为 replay 保留原 prompt，直接违反 NFR-O2；若全脱敏，NFR-O3 acceptance 又无法通过。  
**Suggested fix**: 明确拆成两层：默认 `TraceEvent.payload` 永远脱敏；原始 prompt/stdout 只能进入 `artifact_refs` 指向的 0600 文件，并且 `trace replay` 默认展示脱敏版，显式 `--unsafe-raw` 才读原文。

### M2 [MAJOR] Trace event enum 漏掉文档承诺的错误事件

**Where**: `docs/design/REQUIREMENTS.md §C.4 NFR-F1`，`docs/design/FRAMEWORK.md §5 TraceEventSchema.event_type`  
**Lens**: 1, 2  
**Issue**: NFR-F1 要求 Orchestrator 写 trace `driver.exited.error`，但 `TraceEventSchema.event_type` 只有 `driver.exited`，没有 `driver.exited.error`。  
**Why this matters**: 后续测试如果按 NFR 写，会期望一个 schema 不允许的事件值；如果按 schema 写，又无法满足 NFR。  
**Suggested fix**: 二选一：要么把 NFR-F1 改成 `event_type=driver.exited + status=error`；要么在 enum 加 `driver.exited.error`。更建议前者，因为已有 `status` 字段。

### M3 [MAJOR] Session 模型无法表达 CLI / HTTP / WS 共享同一 session

**Where**: `docs/design/REQUIREMENTS.md §B HTTP/WebSocket 多入口 + session_id 共享`，`docs/roadmap/ROADMAP.md §4 M1A-3`，`docs/design/FRAMEWORK.md §8 SessionSchema.source`  
**Lens**: 2  
**Issue**: `SessionSchema.source` 是单个 enum，但 M1A 明确要求 CLI / HTTP / WS 可以共享同一个 `session_id`。一个 session 后续跨入口使用时，单一 `source` 会变成不准确字段。  
**Why this matters**: iOS / Web / CLI round-trip 时，后端和 Swift Codable 可能都能 decode，但语义会错：session 到底属于哪个入口无法稳定判断。  
**Suggested fix**: 改为 `createdFrom: Source` + `activeSources?: Source[]`，或把入口来源放到每条 `Intent.source`，Session 只保存 `id / instanceId / createdAt / lastActiveAt`。

### M4 [MAJOR] Permission trace 要记录原始敏感路径，和脱敏规则冲突

**Where**: `docs/design/REQUIREMENTS.md §C.3 NFR-P1 / NFR-P3`，`docs/design/REQUIREMENTS.md §C.1 NFR-O2`  
**Lens**: 4  
**Issue**: NFR-P1 要求 trace payload 含 `payload.path=/etc/passwd`，NFR-P3 要求含 `payload.path=~/.ssh/`；但 NFR-O2 说私人路径要脱敏。  
**Why this matters**: `~/.ssh/` 这类路径本身就是敏感信息。权限系统拒绝了访问，却又把敏感路径写进 trace，会破坏“拒绝越权”的安全收益。  
**Suggested fix**: 规定 permission event payload 只保存 `path_class`、`decision`、`capability`、`path_hash`、可选 `redacted_path`，例如 `~/.ssh/id_rsa` 只显示 `~/.ssh/<redacted>`。

### M5 [MAJOR] `Agent.run()` 返回类型让成功/失败状态边界不清

**Where**: `docs/design/FRAMEWORK.md §2.1 Agent.run(intent): Promise<Artifact | AgentResult>`  
**Lens**: 1  
**Issue**: `run()` 可以直接返回 `Artifact`，也可以返回 `AgentResult`。这样调用方需要猜测“直接 Artifact”是否等价于 `{ status: 'success', artifact }`。  
**Why this matters**: cancel / paused / failed 是 Agent 生命周期核心状态。返回 union 会让 Orchestrator、HTTP endpoint、Swift DTO 映射更容易漏掉状态分支。  
**Suggested fix**: 统一返回 `Promise<AgentResult>`，成功也必须是 `{ status: 'success', artifact }`。
