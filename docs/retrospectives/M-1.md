## 5. 学到了什么（沉淀进 LEARNINGS / SKILL）

`.claude/skills/reviewer-cross/LEARNINGS.md` 累积 3 条复用规则：

1. **协议层 ID 格式契约要么 UUIDv4 要么 opaque stable string，但全文必须一致**——schema 注释、Zod 类型、fixture 实际值、Swift String 类型必须给同一概念同一约束
2. **WS event payload 必须复用 DTO 的 enum schema，不允许放宽到 z.string()**——否则 UI 先收 event 再刷 DTO 时两端 decode 都成功但状态值非法
3. **跨端 round-trip 不变量必须真有 Swift 端测试**——含 .nullable / Codable union / AnyCodable 类高风险结构必须 M1+ Swift 实测，不能默认"等下一里程碑"

`~/.claude/skills/debate-review/log.jsonl` 累积 3 条评审历史（含 contract #1+#2 / #2 / #3+#4+method 各一条 JSON 行）。

**作者级反向挑战**（沉淀给下次评审）：
1. **reviewer-cross 缺"里程碑出口自检 lens"**：cross 把 PROTOCOL §8 撒谎只标 minor，arch 标 BLOCKER。下次评审 SKILL 可能要补 lens 6
2. **arch 推荐方案 A user_version gate 在多 migration 仍脆**——选方案 B schema_migrations 表 + 事务包裹更工程化
3. **AuditLog null vs missing 在 TS Zod .nullable() 与 Swift Codable 默认 nil-key drop 之间会破 round-trip**——必须 Swift 端显式 encodeNil
4. **PROTOCOL §8 双向撒谎反 pattern**：[x] 谎称完工 / [ ] 谎称未起步 都需 verify 脚本守门
5. **git-guard `--no-verify` 阻止承诺其实不可 enforce**——降级为 dev guardrail，真守门转移到 layer 1/3/4
6. **ContextBundle grep worktree 等价读非 Bundle 文件**——materialize Bundle 到只读目录是唯一可执行边界
7. **大批文件评审应拆任务**，否则 stream timeout（arch retry 教训）
8. **多 reviewer 独立 lens 拼合才有完整修复**：reviewer-cross 找 grep 漏洞 + arch 找 mayInclude 矛盾，单独都不完整
