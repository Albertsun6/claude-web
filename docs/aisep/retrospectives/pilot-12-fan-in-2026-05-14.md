## What worked

1. **Sharing the bottom-up build order** with the user mid-way: state-machine → workspace-lock → affects-overlap → runner upgrade → runRetryChild → runFanInChildren made each slice atomically reviewable. No "big-bang" PR or stuck rollback.
2. **Q3 revocation** (no `predecessorIds[]` plural field; single-id chain works). Made fan-in mirror linkage trivial — just one field per child.
3. **Decision 4 retry-marker as caller-marker** (not an enum, not a permanent state-machine change). Strict baseline still in force for non-retry callers; only `runRetryChild` ever passes the marker. 0 risk of accidental terminal-status escape elsewhere.
4. **scheduler.nextReadyFanInDispatch as a separate function** (Q2 candidate B). Pure, easy-to-test, 24 tests covering all priority orderings (running > pending > failed > migrated). Runner code reads cleanly.
5. **MockStageExecutor extension** (`failFirstAttemptOnSubStages`) was the smallest possible change to enable retry-child dogfood. The retry-child happy-path test is exactly one extra option toggle.
6. **`literalAnchors` ≥ 3-char filter** in affects-overlap caught a real false-positive risk: too-short patterns like `^a/` were getting through. Plan-validator now rejects them at parse time; existing test fixtures all updated to `^pkg-a/`-style names with a single sed-style replace.

## What surprised

1. **macOS O_CREAT ≠ Linux O_CREAT.** Hardcoded POSIX `O_CREAT = 64` works on Linux but is wrong on Darwin (`O_CREAT = 512`). Workspace-lock initially failed all 6 acquire tests with ENOENT. Fix: use `fs.constants.O_CREAT` etc. — one-line change, but a sharp reminder to never assume POSIX numerical values are portable.
2. **`runFanOutParent` signature change cascaded farther than expected.** Updating `children: Array<{ name }>` → `Array<{ name, affects }>` broke 7 existing tests in `runner-fanout.test.ts` plus the production `commands/run.ts` path. Solution: thread affects through everywhere + auto-generate `packages/<name>/.*` fallback when CLI `--children` flag is used without plan.md backing. Affects also needs to propagate from `parsePlanParallel` → `effectiveAffectsByName` map.
3. **The ADR-022 "baseline 366" doesn't match `pnpm -r test` count.** Actual pre-Slice-1 baseline was 342. The 366 number in the ADR may include some tests that aren't in the `pnpm -r test` path. Used relative-increment math (`+N tests`) instead of absolute targets — see also Slice 1 commit msg.
4. **Q5 ¶7 "parent stays at original status until all children terminal again"** is subtle. After retry-success, parent does NOT flip from `failed → succeeded`. Downstream fan-in dispatch is gated on **children states**, not parent.status. `scheduler.nextReadyFanInDispatch` step 1 accepts `parent.status === "failed"` explicitly for this path. Easy to mis-read as a state-machine bug; the runRetryChild test "parent.status stays 'failed' after retry-success (Q5 ¶7)" pins it down.

## Decisions revisited

- **Decision 1 (β stage-pair fan-in only)**: held. Per-child `predecessorId` is enough; α decoupled fan-in would have ballooned state-machine surface. Slice 4's `runFanInChildren` is ~150 LOC because β makes the structure obvious.
- **Decision 2 (`affects: string[]` required on child)**: held. Some real friction in updating existing tests, but no friction in actual semantics. The `migratedFromV03` audit marker pays for itself by preventing fresh dispatch from migrated workspaces (covered by Slice 3 affects-overlap + Slice 2 scheduler tests).
- **Decision 3 (protocol@0.4.0 MAJOR-class with MINOR label)**: held. ADR-006 supersede language in version.ts comment is clear. Slice 5 migrate utility ships in the same release as required by Decision 5.
- **Decision 4 (id-stable retry + state-machine amendment)**: held. The `{retryChild: true}` caller marker pattern keeps the strict baseline default and only the one authorized caller path (`runner.runRetryChild`) escapes it.
- **Decision 5 (stabilization-period supersede)**: held. `aisep migrate --to 0.4` co-shipped + cross-version round-trip A/B verified at both protocol + binary layers.
- **Q1b whitelist (`{implement, verify, review}`)**: held. `integrate` excluded — confirmed it works as the fan-in terminal aggregation single-row in gate 1 dogfood chain.
- **Q3 revocation (no `predecessorIds[]`)**: held — `stage.ts:120-121` comment updated.
- **Q4 (declared-overlap regex-intersect heuristic)**: mostly held. The ≥ 3-char literal-anchor filter caught one fixture issue (`^a/` short anchors) but matches the spec's "share a common literal-substring anchor" intent. Plan-validator at the CLI layer + runtime `assertNoAffectsOverlap` at the runner layer provide defense-in-depth.
- **Q6 (stacked-timeline visualization)**: partially. `AisepReportParallelGroup.direction` + per-child affects table land; full Mermaid Gantt sub-bars deferred.
